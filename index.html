Introduction

Performance and scalability testing is essential to ensure that the software can handle the intended workload and maintain acceptable performance under heavy workloads. Performance testing is a systematic process to measure the software's ability to perform tasks under different conditions and to evaluate its ability to handle large numbers of users or transactions.

Performance testing focuses on measuring the speed, capacity, stability and scalability of the software under different load conditions, such as number of concurrent users, amount of data processed, response time and error rate. Scalability testing focuses on measuring how the software can handle an increase in workload and how the software can be scaled to handle a larger load.

What is the endurance test?

The word Endurance means 'the ability of something to last or resist wear and tear'. You can simply call it durability, lasting power, or longevity.
When it comes to the world of software, the endurance test (soak test or longevity test or capacity test) It is a type of non-functional test that is performed to verify whether the software system can withstand an expected high load. continued for a long period of time.

  • Why perform resistance tests?

    The main purpose of performing this test is to identify potential memory leaks. So during this test,memory utilization is closely monitored A memory leak is a failure in a software program to free discarded memory, resulting in a poor performance or failure A memory leak may not have an impact in the short term, but in the long term, it slows down the system, which leaves no free memory and thus ultimately leads to application or system crash.

  • Duration of a resistance test

    If it is a 24/7 non-stop operation project, Ideally, an endurance test should be carried out for 72 hours.

  • Benefits of Endurance Testing

    Ensures the suitability of an application, Makes the app robust, checks the sustainability of the system over time.

  • Challenges in endurance tests

    It is one that consumes a lot of time, cannot be done manually, it is often difficult to determine how much load is worth applying, client observes unhandled exceptions.

  • Steps to perform this test

    Set up test environment, Creation of a test plan and test scenarios,Estimated test cycle, Risk analysis, Test program, Execution of the test, Closing of the test cycle

What are scalability tests?

Scalability tests analyze how the system scales up. It indicates, for example, how many more users we can support by adding another application server, or time improvements when we add more CPU to the database server. This type of performance testing should be performed during the development cycle or before the product release, in order to detect problems before the end user himself.

  • Objective of scalability testing

    The objective of scalability testing is: To determine how the application scales with increasing workload, To determine the user limit for the software product, To determine client-side degradation and end-user experience under load, To determine the robustness and server-side degradation.

  • Applications for performance testing

    The most common applications for stress testing are Apache JMeter, LoadRunner, Gatling, among others.

  • Why use Gatling

    Gatling is an open source performance and load testing tool that can be used to evaluate the performance of web applications. Among the advantages of using Gatling for testing are: Scalability, intuitive and easy-to-use, detailed reports

Conclusion

It is important to note that performing a web page stress test can be resource intensive and can affect server and network performance. Therefore, it is recommended to perform these tests in a controlled environment and off-peak hours.
